하루 아침에 프로젝트 생성이 안되는 오류가 발생했다.

최근에 한 일이라곤 Node.js 버전을 업데이트 한 것 밖에 없는데, 이게 문제인줄 모르고 VSCode 터미널에서 프로젝트 생성 명령어가 실행 되지 않아 PowerShell에 설치 명령어를 실행 시켜봤다.

--- 

### VSCode

- VSCode 터미널에서는 오류 메세지 **확인 불가**

### PowerShell

- PowerShell에서는 오류 메세지 **확인 가능**

오류 메세지 내용은 대략 **PowerShell의 정책으로 인한 문제** 라는 것이었다.

---

> 지금 생각해보니 이 오류와 내가 해결하고자 하는 프로젝트 생성이 되지 않는 오류와는 다른 문제였다.

## 해결 방법

1. 관리자 권한으로 PowerShell 실행

- Window 키 + X -> PowerShell 또는 터미널 실행

2. 실행 정책 확인

```
Get-ExecutionPolicy
```

- 보통 **Restricted** 또는 **AllSigned**로 설정되어 있으면 스크립트 실행이 차단됨

3. 실행 정책을 일시적으로 변경

```
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

- Y 입력 후 엔터

4. 다시 실행

```
명령어 다시 실행
```

---

이 방법으로 PowerShell에 뜨는 오류는 해결됐지만 _여전히 프로젝트 생성이 되지 않았다._

이후에 여러 방법을 고안해보고 적용해봤지만 최초의 오류는 해결되지 않았고, 마지막 최후의 방법으로 Node.js를 다운그레이드 해보았다.

결과는 성공...하지만 원인을 아직도 모르겠다🫠 대충은 버전 충돌로 인한 문제같은데 이틀내내 여기에 시간을 할애했더니 잠깐 동안은 쳐다보기도 싫어졌다.
